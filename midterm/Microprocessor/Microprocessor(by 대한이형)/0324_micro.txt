 프로세서가 레지스터를 내장하도록 설계하는 것이 [폰 노이만]의 컴퓨터의 기본 구조.

MAR (Memory Address R)
: 메모리의 주소값 (주소값) -> 주소 버스로
MBR (Buffer): 메모리 데이터 레지스터
-> 데이터 버스
+ 주소와 데이터는 따로 논다.

IR (instruction 명령): 명령 레지스터
PC (Program Counter): 다음에 실행될 명령어의 주소값이 들어감 (ex: c언어 소스코드에서 한 줄을 가져오면 다음 줄이 있을테니. 그 다음 줄의 정보)

ALU: 여러 계산을 동시에 수행 가능

ALU에서 연산을 한다음의 연산의 결과를 AC 레지스터에 (누산기) 저장. 계산값 (새로운 데이터)를 CU (제어장치) 에서 어떤 명령이 내려오냐에 따라 저장 값 처리가 달라짐

  -레지스터-메모리 명령
· 레지스터 -> 메모리 : 스토어
· 메모리 -> 레지스터: 로드
  -레지스터 레지스터 명령
A + B : A와 B (연산 대상) = 오퍼랜드
-> 결과 값을 AC에 저장

   4. 프로세서 명령 실행
: 인출(1, 2) fetch - 해독(3, 4) decode -  실행(5, 6) execute 사이클로 구성

*해독기 (microprogrammed control): 하드웨어를 소프트웨어로 대체

*산술 논리 연산 장치(ALU): 산술 연산과 논리 연산
 ·산술: (덧,뺄,곱,나눗)셈, 증가, 감소, 보수
 ·논리: AND, OR, NOT, XOR, 시프트

 상태 레지스터
· C: Carry (올림수)
· Z: Zero
· N: Negative
· V: oVerflow
· P: Parity
